# 需求分析文档

## 1. 项目背景

### 1.1 项目名称
**给最爱的你** - 女朋友专属浪漫网站

### 1.2 项目目标
为女朋友创建一个专属的浪漫网站，用于记录两人之间的美好时光、心情变化、重要时刻，并提供情绪宣泄的出口。

### 1.3 目标用户
- **主要用户**：女朋友（日常使用）
- **次要用户**：自己（维护和更新内容）

### 1.4 使用场景
- 日常心情记录
- 情绪宣泄和吐槽
- 重要日子倒计时
- 照片回忆存储
- 重要事件记录
- 智能聊天互动

---

## 2. 功能需求

### 2.1 心情记录模块

#### 功能描述
记录每天的心情状态，支持多种心情表情和文字描述。

#### 输入项
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 心情选择 | 单选 | 是 | 6种预设表情（开心、爱你、难过、生气、想你、困了） |
| 心情文字 | 文本 | 是 | 心情描述，最多500字 |

#### 输出项
- 心情历史列表
- 按时间倒序展示
- 显示心情表情、文字、时间

#### 业务规则
1. 必须选择心情表情才能提交
2. 必须填写心情文字才能提交
3. 心情记录按时间倒序排列
4. 支持删除心情记录

#### 数据结构
```javascript
{
    id: timestamp,          // 唯一标识
    emoji: "😊",           // 心情表情
    mood: "happy",         // 心情类型
    text: "今天很开心",     // 心情描述
    time: "2024/1/1 12:00" // 记录时间
}
```

---

### 2.2 吐槽/骂人模块

#### 功能描述
提供情绪宣泄的出口，支持多种吐槽类型和强度控制。

#### 输入项
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 吐槽类型 | 单选 | 是 | 3种类型（吐槽、骂人、抱怨） |
| 吐槽内容 | 文本 | 是 | 吐槽内容，最多1000字 |
| 生气程度 | 滑块 | 是 | 1-10级，表示生气程度 |

#### 输出项
- 吐槽记录列表
- 显示类型标签、内容、强度指示条
- 按时间倒序展示

#### 业务规则
1. 必须选择吐槽类型才能提交
2. 必须填写吐槽内容才能提交
3. 生气程度默认5级
4. 支持删除吐槽记录

#### 数据结构
```javascript
{
    id: timestamp,          // 唯一标识
    type: "rant",          // 类型：rant/scold/complain
    typeName: "吐槽",      // 类型名称
    text: "太气人了",       // 吐槽内容
    intensity: 8,          // 强度 1-10
    time: "2024/1/1 12:00" // 记录时间
}
```

---

### 2.3 倒计时模块

#### 功能描述
记录重要日子的倒计时，支持多个倒计时同时显示。

#### 输入项
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 事件名称 | 文本 | 是 | 倒计时事件名称 |
| 目标日期 | 日期时间 | 是 | 倒计时目标日期时间 |
| 事件图标 | 单选 | 是 | 7种预设图标（生日、纪念日、节日等） |

#### 输出项
- 倒计时卡片列表
- 实时更新天、时、分、秒
- 显示事件名称、图标、目标日期

#### 业务规则
1. 必须填写事件名称和目标日期才能添加
2. 支持添加多个倒计时
3. 倒计时实时更新（每秒刷新）
4. 支持删除倒计时
5. 目标日期必须是未来时间

#### 数据结构
```javascript
{
    id: timestamp,              // 唯一标识
    title: "生日",              // 事件名称
    date: "2024-12-31T00:00",  // 目标日期
    icon: "🎂"                 // 事件图标
}
```

---

### 2.4 相册模块

#### 功能描述
存储和展示两人的照片，支持照片上传和故事描述。

#### 输入项
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 照片 | 文件 | 是 | 支持图片格式（jpg、png、gif等） |
| 照片描述 | 文本 | 否 | 照片的故事或描述，最多200字 |

#### 输出项
- 瀑布流相册展示
- 照片卡片（图片、描述、时间）
- 点击查看大图

#### 业务规则
1. 支持拖拽上传和点击上传
2. 支持同时上传多张照片
3. 照片以Base64格式存储
4. 支持删除照片
5. 点击照片可查看大图

#### 数据结构
```javascript
{
    id: timestamp,              // 唯一标识
    src: "base64...",          // 图片数据
    caption: "美好回忆",       // 照片描述
    time: "2024/1/1 12:00"    // 上传时间
}
```

---

### 2.5 时间轴模块

#### 功能描述
记录两人的重要事件和里程碑，以时间轴形式展示。

#### 输入项
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 事件日期 | 日期 | 是 | 事件发生的日期 |
| 事件标题 | 文本 | 是 | 事件标题，最多50字 |
| 事件描述 | 文本 | 否 | 事件详细描述，最多500字 |
| 事件图标 | 单选 | 是 | 8种预设图标（相爱、庆祝、旅行等） |

#### 输出项
- 垂直时间轴展示
- 事件卡片（图标、日期、标题、描述）
- 按日期倒序排列

#### 业务规则
1. 必须填写事件日期和标题才能添加
2. 时间轴按日期倒序排列
3. 支持删除时间轴事件
4. 左侧时间线，右侧事件卡片

#### 数据结构
```javascript
{
    id: timestamp,              // 唯一标识
    date: "2024-01-01",        // 事件日期
    title: "第一次见面",       // 事件标题
    desc: "在咖啡厅...",       // 事件描述
    icon: "💕"                // 事件图标
}
```

---

### 2.6 智能助手模块

#### 功能描述
模拟男友回复，陪女朋友聊天解闷，帮用户回答女朋友的问题。

#### 交互方式
- 聊天窗口形式
- 支持文字输入
- 实时回复

#### 输入项
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 消息内容 | 文本 | 是 | 用户输入的消息 |

#### 输出项
- 聊天消息列表
- 用户消息（右侧，粉色背景）
- 助手回复（左侧，白色背景）

#### 业务规则
1. 基于关键词匹配的智能回复
2. 支持多种话题：
   - 日常问候（你好、早安、晚安）
   - 表达爱意（爱你、想你、喜欢你）
   - 情绪安慰（难过、生气、累了）
   - 常见问题（你爱我吗、我漂亮吗）
3. 回复内容温馨浪漫
4. 支持多条回复随机选择

#### 回复示例
```javascript
{
    "你好": ["你好呀！今天也要开心哦~ 💕", "嗨！想我了吗？🥰"],
    "爱你": ["我也爱你！超级超级爱你！❤️❤️❤️", "爱你哦！永远爱你！💕"],
    "难过": ["怎么了？告诉我，我陪你~ 🥺", "抱抱！别难过了，有我在呢！💕"]
}
```

## 3. 非功能性需求

### 3.1 用户体验
- **响应式设计**：支持手机、平板、桌面端
- **流畅动画**：页面切换、元素出现动画
- **温馨浪漫**：粉色主题，爱心元素
- **直观导航**：固定顶部导航，快速跳转

### 3.2 数据存储
- **存储方式**：浏览器 localStorage
- **数据持久化**：刷新页面不丢失
- **数据容量**：建议不超过5MB（避免性能问题）

### 3.3 性能要求
- **页面加载**：< 2秒
- **动画流畅**：60fps
- **图片优化**：压缩后上传

### 3.4 兼容性
- **浏览器**：Chrome 60+、Firefox 55+、Safari 11+、Edge 79+
- **设备**：手机、平板、桌面电脑

### 3.5 安全性
- **数据本地存储**：不上传到服务器
- **隐私保护**：仅本地访问

## 5. 约束条件

### 5.1 技术约束
- 纯前端实现，无后端服务
- 使用浏览器 localStorage 存储数据
- 不依赖外部框架（原生 HTML/CSS/JS）

### 5.2 时间约束
- 开发周期：1-2周
- 优先实现核心功能

### 5.3 资源约束
- 个人开发，无团队支持
- 使用免费资源（图标、字体等）

## 7. 风险评估

### 7.1 技术风险
- **风险**：localStorage 容量限制
- **缓解**：定期清理旧数据，压缩图片

### 7.2 用户体验风险
- **风险**：动画过多导致卡顿
- **缓解**：优化动画性能，减少复杂动画

### 7.3 数据安全风险
- **风险**：清除浏览器数据丢失记录
- **缓解**：提示用户定期备份
